Q: Can I add a database?
A: NO. Never. The system is designed for in-memory state only.

Q: Will data persist after server restarts?
A: NO. It's intentional. This allows you to migrate to a real DB later.

Q: Can users auto-withdraw money?
A: NO. Admin must approve every withdrawal manually.

Q: Can I auto-settle bets?
A: NO. Admin must settle every bet manually.

Q: How many users can the system handle?
A: Limited by server RAM. Render free tier has enough for testing.

Q: Can I use the same Vercel project for frontend and admin?
A: NO. They must be separate projects in Vercel.

Q: What if I need to add a real database?
A: The models/services are designed for easy DB migration.

Q: How do I update code after deployment?
A: Just push to GitHub. Render and Vercel auto-deploy.

Q: Where do I store API keys?
A: In .env file on each platform (never in code).
